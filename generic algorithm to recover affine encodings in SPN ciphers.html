CEJO framework

Basic strategy: the whole cipher is decomposed into round functions and the round functions are represented by summation of lookup tables with small size.

Chow et al.'s strategy (a white-box AES implementation and a white-box DES implementation) provided a framework, called "CEJO framework", for designing white-box implementation of using table lookups.

Conclusion

  1. As we can see from previous implementations, it is very difficult to design a white-box implementation with a security level similar to the black-box model.
    Hence, the practical objective of white-box implementations is to increase the complexity of cryprtanalysis.

  2. All of the implementations mentioned above suffered unpredicted attacks soon after their designs were announced.
    This is mainly because there are no standard attack tools such as differential cryptanalysis and linear cryptanalysis for block ciphers.

Introduction of the Generic Algorithm 2

For solving the affine equivalence problem in the case where the inner non-linear layer is composed of parallel S-boxes.

Our algorithm solves the following problem:

Problem 1

Let FFF be an nnn-bit to nnn-bit permutation such that F=B∘S∘AF = B \circ S \circ AF=B∘S∘A, where:

  1. AAA and BBB are nnn-bit affine layers;

  2. S=(S1,⋯ ,Sk)S=(S_{1}, \cdots, S_{k})S=(S1​,⋯,Sk​) consists of the parallel application of kkk permutations SiS_{i}Si​ on mmm bits each (called S-boxes). Note that n=kmn=kmn=km.

Knowing SSS, and given oracle access to FFF (but not F−1F^{-1}F−1), find affine A′A'A′, B′B'B′ such that F=B′∘S∘A′F = B'\circ S \circ A'F=B′∘S∘A′.

Remark 1

Our statement of the problem allows the algorithm to query FFF, but not F−1F^{-1}F−1.

access to FFF, but not to F−1F^{-1}F−1

CEJO framwork
The output of FFF is computed as a sum of sum hard-coded table outputs, and inverting FFF would require knowing how to split a given output of FFF into the appropriate sum.

Baek et al. also propose an algorithm when only FFF is accessible, but it is much slower.

access to both FFF and F−1F^{-1}F−1

Our own algorithm:

  1. isolate the input and output space of each S-box

  2. exhaust that space in 2m2^m2m operations for each S-box, which will allow us to access the inverse mapping of each S-box.

Essentially, our algorithm will allow us to revert back to the case where the direct and inverse mappings are both available.

In particular, it is not obvious how our algorithm could be improved even if F−1F^{-1}F−1 were accessible.

Baek et al. explicitly provide an algorithm to solve Problem 1 when FFF and F−1F^{-1}F−1 are both available, in O(n423m/m)O(n^42^{3m}/m)O(n423m/m) operations.
However this is slower than our algorithm for all reasonable parameter ranges, even though our algorithm does not require access to F−1F^{-1}F−1.

Remark 2

Problem 1 asks to recover some affine encodings A′A'A′, B′B'B′ such that F=B′∘S∘A′F=B'\circ S\circ A'F=B′∘S∘A′, but not necessarily AAA and BBB.

Reason: AAA and BBB may not be uniquely defined.

(A,B)→(P∘A,B∘P−1)(A,B)\rightarrow (P \circ A, B\circ P^{-1})(A,B)→(P∘A,B∘P−1), where PPP is any permutation swapping S-box inputs.

Problem 1 merely asks to recover a solution.

Use the algorithm by Dinur to solve the Problem 1, that algorothm is able enumerate all solutions if desired, it is straightforward to adapt our algorithm so that it outputs every solution.

Remark 3

The special case: encodings are linear instead of affine

Our algorithm eventually reduced Problem 1 to the affine equivalence problem for each S-box separately.

Using a linear equivalence algorithm on each S-box, instead of an affine one.

Remark 4

The special case: k=1k=1k=1, i.e. SSS is composed of a single S-box.

Practical complexity for nnn upwards of 128 bits, by using the fact that SSS is split into relatively small mmm-bit S-boxes.
